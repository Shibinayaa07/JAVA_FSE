Spring Core and Maven Exercises: Step-by-Step Instructions

Exercise 1: Configuring a Basic Spring Application
1. Create a Maven project named `LibraryManagement`.
2. Add Spring Core and Spring Context dependencies to `pom.xml`.
3. Create packages:
   - `com.library.service`
   - `com.library.repository`
4. Create `BookService` in the service package.
5. Create `BookRepository` in the repository package.
6. Add `applicationContext.xml` inside `src/main/resources`.
7. Define beans for `BookService` and `BookRepository` in the XML file.
8. Create a main class to load the Spring context.
9. Run the application and verify that the context loads without errors.

Exercise 2: Implementing Dependency Injection
1. Add a `BookRepository` field in `BookService`.
2. Create a setter method for that field.
3. Update `applicationContext.xml` to wire `BookRepository` into `BookService`.
4. Add a method in `BookService` that uses the repository.
5. Run the main class and confirm that the dependency is injected correctly.

Exercise 3: Implementing Logging with Spring AOP
1. Add Spring AOP and AspectJ Weaver dependencies to `pom.xml`.
2. Create the package `com.library.aspect`.
3. Create `LoggingAspect` and mark it as an aspect.
4. Add advice to log method execution time.
5. Enable AspectJ support in `applicationContext.xml`.
6. Register the aspect or enable component scanning.
7. Run the application and confirm timing logs appear in the console.

Exercise 4: Creating and Configuring a Maven Project
1. Create or open the `LibraryManagement` Maven project.
2. Add dependencies for Spring Context, Spring AOP, and Spring WebMVC.
3. Configure the Maven Compiler Plugin for Java 1.8.
4. Refresh the Maven project in your IDE.
5. Build the project and confirm that dependency resolution succeeds.

Exercise 5: Configuring the Spring IoC Container
1. Create `applicationContext.xml` in `src/main/resources`.
2. Define the `BookService` bean.
3. Define the `BookRepository` bean.
4. Inject `BookRepository` into `BookService`.
5. Load the application context in the main class.
6. Retrieve the bean and call its methods to test the container.

Exercise 6: Configuring Beans with Annotations
1. Enable component scanning for `com.library`.
2. Annotate `BookService` with `@Service`.
3. Annotate `BookRepository` with `@Repository`.
4. Use `@Autowired` where dependency injection is needed.
5. Remove duplicate XML bean definitions if annotation scanning is used.
6. Run the program and verify that annotation-based configuration works.

Exercise 7: Implementing Constructor and Setter Injection
1. Add a constructor in `BookService` that accepts `BookRepository`.
2. Keep or add a setter for the repository if setter injection is required.
3. Configure constructor injection in XML using `<constructor-arg>`.
4. Configure setter injection using `<property>` if needed.
5. Load the bean in the main class and test both injection styles.

Exercise 8: Implementing Basic AOP with Spring
1. Create `LoggingAspect` in `com.library.aspect`.
2. Add `@Before` advice to log before method execution.
3. Add `@After` advice to log after method execution.
4. Register the aspect in Spring.
5. Enable auto-proxying with AspectJ.
6. Call a target service method from the main class.
7. Confirm the aspect is triggered during execution.

Exercise 9: Creating a Spring Boot Application
1. Create a Spring Boot project named `LibraryManagement`.
2. Add Spring Web, Spring Data JPA, and H2 Database dependencies.
3. Configure `application.properties` for H2 and JPA.
4. Create the `Book` entity class.
5. Create `BookRepository` and use `@Repository`.
6. Create `BookController` for CRUD endpoints.
7. Start the application and test the REST endpoints.
8. Verify the application connects to H2 and persists data correctly.

Final procedure to complete all exercises:
1. Complete the Maven setup first.
2. Create the base package structure.
3. Implement XML bean configuration and test the container.
4. Add dependency injection and verify object wiring.
5. Add AOP support and validate logging behavior.
6. Switch to annotation-based configuration and test again.
7. Demonstrate constructor and setter injection.
8. Finally, create the Spring Boot version and test the REST API.
